Arrays vs. Hashes

How Ruby Contains Lists of Values

February 15th, 2016

Ruby has two different containers of data that are called upon regularly and these are arrays and hashes. Both of them can be described as indexed collections with a key and corresponding value, however they each act a little differently as we will see below.

Arrays are an ordered list where each value inside is called upon with a corresponding key that is a numbered index position. The array is displayed in ruby as a list of things, inside a bracket, each value seperated by a comma like so: "array_list = [value1, value2, value3]". The corresponding indexed position begins with 0, so "value1" is actually in the 0 position and not the 1 position. So if you wanted to target just the first value (value1) you would type in "array_list[0]" and then add 1 for the next corresponding position. So then 0 is the array key for "value1". When I say a list of things, the things can be mostly any ruby value such as a string, an integer, a float or even another array.

It can be argued that it is more efficient to use arrays over hashes for the most part but not when you want a little more flexibility with your keys. A hash is displayed very similarly to an array but instead of brackets we use those fancy little curly bracers { }. Also unique to hashes is that the key is actually found inside the bracers along with their corresponding values like so: "basic_hash = {"Name" => "Ryan", "Location" => "California", "Age" => "Nice try"}. So here we see hashes have unique keys defined with unique corresponding values. Once the key is defined, you just have to call upon it's value using the unique corresponding key like you did with arrays using index positions: 'basic_hash["Location"] => "California"'
