When arrays and hashes work together!

February 21st, 2016

In my previous post I talked about the ruby classes Array and Hashes, and how they relate to one another. We figured out which class works best for which situation and we looked at different ways to loops through them to better create methods that solve the problem we're trying to use ruby for. In addition to classes, ruby has modules, which also contain methods and constants. Think of a module as one parent of a class. This means that classes will inherit all the methods (built-in or not) of their parent. One such module whose methods you use constantly is called Enumerable. The Enumerable module, contains most of the methods we use to iterate through classes like Array and Hashes. I'm going to focus on the Enumerable method #group_by because it's an interesting iteration method that actually returns a value that contains both Arrays and Hashes! Arrays and Hashes working together? Who knew! Actually it's pretty good practice using them together to accomplish the same goals, #group_by is just one of the many examples of this.

Simply put, #group_by breaks a collection into groups based on the given criteria and returns a hash. Remember how a hash has both a unique key and value pair? Well the hash that is returned actually has an array as its value. This array of elements should be members of the original collection that correspond with the key group. SO, let's start simple and just say we have a list of numbers and we want to group them using modulus (%):

(0..25).group_by {|number| number % 3}
=> {0=>[0, 3, 6, 9, 12, 15, 18, 21, 24], 1=>[1, 4, 7, 10, 13, 16, 19, 22, 25], 2=>[2, 5, 8, 11, 14, 17, 20, 23]}

Here we get three groups. The first group with key 0 has all the corresponding numbers within the range of 0 to 25 that are evenly divisible by the number 3 with no remainder. Key 1 has all the numbers that when we divide by three there is a remainder of 1 and key 2 has a remainder of 2. Therefore using this example the keys are equal to the remainder and the value is the group of numbers that correspond to the modulus remainders. (More examples and styling to come...)
